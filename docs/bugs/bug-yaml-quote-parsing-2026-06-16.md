# Bug: plan-tracker.py fails to parse YAML frontmatter with quoted status values

## Summary

`shared/hooks/plan-tracker.py` used a regex pattern `r"^status:\s*(\S+)"` to parse the
`status` field from plan frontmatter. This pattern fails when the status value is quoted
(e.g., `status: "active"` or `status: 'active'`), which is valid YAML syntax.

## Symptoms

- Plan files with quoted status values are skipped during `git push` gate checks
- Plans that should block push (because they have pending TODO items) are incorrectly
  allowed to pass
- No error or warning is emitted

## Root Cause

The `STATUS_PATTERN` regex uses `\S+` to match the status value. This matches:
- `status: active` → extracts `active` ✓
- `status: "active"` → extracts `"active"` ✗ (includes quotes)
- `status: 'active'` → extracts `'active'` ✗ (includes quotes)

When quotes are included, the comparison `status != "active"` evaluates as true even
when the plan is actually active.

## Impact

- Plans with quoted status values are silently ignored
- Git push gate becomes unreliable for these plans
- Users may inadvertently push incomplete work

## Fix

Update `STATUS_PATTERN` to handle optional quotes:

```python
STATUS_PATTERN = re.compile(r'^status:\s*["\']?([^"\'\s]+)["\']?', re.MULTILINE)
```

This pattern:
- Matches optional opening quote: `["\']?`
- Captures the status value (excluding quotes): `([^"\'\s]+)`
- Matches optional closing quote: `["\']?`

## Verification

Created test case `test_plan_with_quoted_status` in `test_plan_tracker.py`:

```python
def test_plan_with_quoted_status(self):
    (self.plans_dir / "quoted.md").write_text(
        '---\nstatus: "active"\n---\n# Plan\n- TODO: task 1\n'
    )
    result = subprocess.run(
        [sys.executable, self.tracker_script, str(self.repo_root)],
        capture_output=True,
        text=True,
    )
    self.assertEqual(result.returncode, 1)  # Should block push
    self.assertIn("TODO: task 1", result.stdout)
```

Test passes after fix (exit code 1 when active plan has pending TODOs).
